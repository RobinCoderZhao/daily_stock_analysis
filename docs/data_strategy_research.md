## 一、数据获取层评估

### 1.1 当前架构总览

系统采用**策略模式 + 自动故障切换**，共 6 个数据源：

| 数据源 | 优先级 | 类型 | 数据范围 | 稳定性 |
| --- | --- | --- | --- | --- |
| EfinanceFetcher | P0 | 东方财富 | A股全量 + 实时 | ⚠️ 有封禁风险 |
| AkshareFetcher | P1 | 多源封装 | A股/港股/ETF | ⚠️ 接口频繁变动 |
| TushareFetcher | P0/P2 | Pro API | A股全量 | ✅ 稳定（需Token） |
| PytdxFetcher | P2 | 通达信 | A股日线 | ✅ 较稳定 |
| BaostockFetcher | P3 | 学术级 | A股历史 | ✅ 稳定但数据更新慢 |
| YfinanceFetcher | P4 | Yahoo | 美股/兜底 | ⚠️ 海外部署需代理 |

**亮点：**
- 熔断器模式（CircuitBreaker）：连续失败 3 次自动熔断，5 分钟冷却
- 实时行情字段补充机制：主数据源缺字段时自动从次源补齐（volume_ratio、turnover_rate 等）
- 批量预取优化：≥5 只股票自动触发全量缓存

### 1.2 发现的数据问题

#### 🔴 问题 1：turnover_rate 缺失率高

`get_turnover_profile` 依赖 `turnover_rate` 列，但**只有 efinance 和 akshare_em 两个全量数据源**在日线数据中包含换手率。当这两个源被熔断或不可用时，pytdx / baostock 返回的日线数据**不含 turnover_rate**，导致：
- `analyze_trend` 的 turnover 相关字段全部为 None
- `get_turnover_profile` 返回 error
- emotion_cycle 策略完全无法执行

```
影响策略：emotion_cycle, bottom_volume, shrink_pullback
严重程度：高 — 数据缺失导致策略判断失效
```

**建议修复：** 在 DataFetcherManager 层添加 turnover_rate 回填逻辑：
1. 日线数据获取后检测 turnover_rate 是否存在
2. 如不存在，尝试从实时行情的 turnover_rate 字段获取当日值
3. 对于历史数据，通过 `volume / 流通股本 * 100` 公式近似计算

#### 🔴 问题 2：北向资金数据中断

2024 年 8 月起沪深交易所调整了沪港通/深港通信息披露机制，不再实时披露单日买入/卖出金额。akshare 的 `stock_hsgt_hist_em` 接口**数据已不完整**。

```
影响：依赖外资动向的分析缺少关键输入
严重程度：中 — 影响趋势判断的一个维度
```

**建议：** 改用收市后总额数据或 tushare 的北向资金接口作为替代源。

#### 🟡 问题 3：efinance 高频爬取被封禁

当同时分析大量股票（>20 只）时，efinance 的东财接口可能触发 IP 限流（ServerDisconnectedError），导致：
- 所有使用 efinance 的请求连续失败
- 触发熔断器，5 分钟内全部降级到 akshare
- akshare 同样可能被限流，形成**连锁熔断**

```
影响：批量分析场景下数据获取可靠性大幅下降
严重程度：中 — 影响定时推送任务
```

**建议修复：**
1. 为 efinance 添加请求间隔（min 200ms），而非依赖 `random_sleep(1-3s)`
2. 引入代理轮换池（针对服务器部署）
3. 优先使用 tushare Pro（如果 Token 可用），其 API 限流更宽松

#### 🟡 问题 4：数据标准化不一致

各 fetcher 返回的 `pct_chg` 字段口径不同：
- efinance：百分比值（如 5.23 表示 +5.23%）
- baostock：小数值（如 0.0523）
- akshare：百分比值但可能含字符串 "涨停"

`_clean_data()` 只做了 `pd.to_numeric(errors='coerce')`，**没有统一口径**。这会导致策略阈值判断（如乖离率 > 5%）在不同数据源下产生不同结果。

```
影响策略：bull_trend（乖离率判断）、所有使用 pct_chg 的逻辑
严重程度：中 — 可能产生错误的"追高"或"不追"判断
```

**建议修复：** 在 `_clean_data()` 中添加 pct_chg 口径统一逻辑：
```python
# Normalize pct_chg to percentage format (5.23 means +5.23%)
if 'pct_chg' in df.columns:
    max_val = df['pct_chg'].abs().max()
    if max_val < 1:  # Likely decimal format (0.0523)
        df['pct_chg'] = df['pct_chg'] * 100
```

#### 🟢 问题 5：Baostock 数据更新延迟

Baostock 的历史数据通常比实际交易日**延迟 1-2 个工作日**更新。如果 efinance 和 akshare 同时被熔断，降级到 baostock 时，最新 1-2 天的数据会缺失。

```
影响：最新走势判断可能基于滞后数据
严重程度：低 — 仅在多源同时故障时触发
```

## 二、策略改进评估

### 2.1 2026 年 A 股量化行业最新动态

> 数据来源：证券时报、财新、华尔街见闻、东方财富等（2026.01-03）

#### 核心趋势

| 趋势 | 描述 | 对本系统的意义 |
| --- | --- | --- |
| **Alpha 获取难度剧增** | 2026 年前两个月量化指增超额收益显著分化，中证500/1000指增出现负超额 | 传统量价因子同质化严重，必须差异化 |
| **高质量因子持续跑赢** | 高质量因子 1-2 月累计跑赢沪深300约 2.4%，成为最稳的 Alpha 源 | 可新增质量因子选股逻辑 |
| **择时策略回归** | 基于逻辑规则的择时（而非机器学习黑箱）重新成为焦点 | 与本系统"趋势交易"理念天然契合 |
| **分域建模** | 头部私募按市值/风格分域独立建模，挖掘局部超额 | 可按板块分域应用不同策略 |
| **财务附注质量因子** | 存货管理因子 + 坏账管理因子在中小市值股中表现优异 | 可作为新的选股维度 |
| **AI 深度学习选股** | Mamba-MoE 等新模型用于周度选股，RankIC 表现突出 | 长期方向，目前超出系统复杂度 |

#### 2026 年的关键教训

1. **不要依赖单一量价因子** — 传统动量/反转因子在 A 股表现不稳，需多因子交叉验证
2. **质量 > 动量** — 高质量因子（盈利稳定性、现金流）2026 年表现远超动量因子
3. **择时不死** — 规则化择时策略（非机器学习）在 2026 年重新被验证有效
4. **分域思维** — 大盘股和小盘股需要不同策略，不能一把梭

### 2.3 推荐新增的策略

#### 🔥 策略 A：KDJ + RSI 超卖共振（推荐 confidence: 0.90）

**原理：** KDJ 和 RSI 同时进入超卖区域后反弹，形成共振买入信号。经典择时策略，2026 年验证仍然有效。

**条件：**
1. RSI(14) < 30（超卖区域）
2. KDJ J 值 < 0 或 K < 20
3. KDJ 金叉（K 上穿 D）
4. 当前处于 MA20 上方（趋势过滤器）

**所需工具改动：** 在 `stock_analyzer.py` 添加 KDJ 计算（RSI 已有）

#### 🔥 策略 B：均线粘合后发散（推荐 confidence: 0.85）

**原理：** MA5/MA10/MA20 三线粘合后向上发散 = 趋势启动。2026 年择时策略回归背景下，此类规则化信号重新被重视。

**条件：**
1. MA5、MA10、MA20 最大价差 < 收盘价的 1%
2. 随后 MA5 开始上穿 MA10、MA20
3. 成交量逐步放大（量比 > 1.2）
4. MA20 斜率由平转上（ma20_direction: up）

**所需工具改动：** 无，现有数据完全支持

#### 🔥 策略 C：ATR 收缩后突破（推荐 confidence: 0.80）

**原理：** ATR 持续收缩 → 波动率压缩 → 市场蓄势。ATR 突然扩大且方向向上 = 趋势爆发。

**条件：**
1. ATR 连续 5 天以上收缩（atr_trend: contracting）
2. 当日 ATR 突然扩大 > 50%
3. 收盘价向上突破（阳线）
4. 成交量放大配合

**所需工具改动：** 无，Phase 1 已有 ATR 数据

#### 🆕 策略 D：高质量因子选股（推荐 confidence: 0.85）— 2026 新策略

**原理：** 基于 2026 年最新研究，高质量因子（盈利稳定性 + 现金流健康度）是当前 A 股最稳健的 Alpha 来源。高质量因子 2026 年前 2 月累计跑赢沪深300 约 2.4%。

**条件：**
1. 最近 4 季度 ROE 均 > 8%（盈利稳定）
2. 经营现金流/净利润 > 0.7（现金流质量）
3. PE(TTM) < 行业中位数（估值不贵）
4. 资产负债率 < 60%（财务安全）
5. 近 3 年营收 CAGR > 5%（可持续增长）

**所需工具改动：** 需新增 `get_fundamental_profile` 工具，获取 ROE、现金流、PE、负债率等基本面数据

#### 🆕 策略 E：板块轮动择时（推荐 confidence: 0.75）— 2026 新策略

**原理：** 基于 2026 年"分域建模"思路。不同板块在不同市场环境下有不同的最优策略。先判断当前强势板块，再在板块内选股。

**条件：**
1. 获取行业板块排名（已有 `get_sector_rankings` 工具）
2. 筛选近 5 日涨幅 Top5 板块
3. 仅在强势板块内应用趋势策略
4. 弱势板块执行"远离"策略

**所需工具改动：** 需增强 `get_sector_rankings` 返回近 5 日连续数据

#### 🆕 策略 F：资金流向驱动（推荐 confidence: 0.70）— 2026 新策略

**原理：** 主力资金持续流入 + 量价配合 = 强势延续。2026 年 A 股市场结构中，机构资金流向仍是重要领先指标。

**条件：**
1. 近 5 日主力资金净流入 > 0
2. 近 5 日成交额持续放大
3. 均线多头排列
4. 筹码分布显示获利盘 50-80%（健康区间）

**所需工具改动：** 需新增 `get_fund_flow` 工具获取主力/散户资金流向数据

### 2.4 低效策略优化建议

#### dragon_head（龙头策略）→ 建议移出默认策略

**原因：**
- 所需数据（板块涨停数、连板高度、市场情绪温度）大部分不在工具链中
- LLM 难以准确判断"龙头地位"，主观性太强
- confidence_weight 仅 0.45，是最低的
- 2026 年主题行情极致演绎使得龙头判断更加困难

#### wave_theory（波浪理论）→ 建议降级为参考

**原因：**
- 波浪计数极度主观，同一走势可有多种数法
- LLM 无法可靠执行（confidence 0.40）
- 2026 年策略同质化衰减报告中，波浪理论类策略表现最差
