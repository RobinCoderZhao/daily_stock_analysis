# RD-Agent 产品与架构文档

> 微软开源 LLM 驱动的自主研发 Agent 深度分析

## 1. 项目概述

**RD-Agent** 是微软研究院开源的 LLM 驱动自主 R&D Agent 框架，旨在自动化数据驱动场景中最关键的研发流程——提出假设（Research）和实现验证（Development）。

| 维度 | 说明 |
|------|------|
| **开源协议** | MIT License |
| **开发语言** | Python 3.10 - 3.11 |
| **核心定位** | LLM 驱动的自主研发 Agent，自动化因子挖掘与模型优化 |
| **支持平台** | 仅 Linux（Docker 依赖） |
| **LLM 后端** | LiteLLM（支持 GPT、Claude、DeepSeek 等） |
| **论文支持** | NeurIPS 2025 接收，多篇 arXiv 论文 |
| **核心场景** | 量化因子挖掘、模型优化、Kaggle 竞赛、通用数据科学 |

## 2. 核心能力

### 2.1 自动化量化因子挖掘（Quant Factor Mining）

RD-Agent 的核心量化场景能力：

- **自主挖掘 Alpha 因子**：LLM 自动提出因子假设，生成 Python 实现代码
- **自动回测验证**：在 Qlib 平台上自动验证因子有效性
- **迭代进化**：根据回测反馈自动优化因子表达式
- **知识积累**：维护因子知识库，避免重复探索

```
因子进化循环：
  提出因子假设 → 实现因子代码 → Qlib 回测验证 → 分析反馈
       ↑                                              │
       └──────────── 知识库更新 ←─────────────────────┘
```

**关键指标**：
- 成本 < $10 即可完成一轮完整因子挖掘
- ARR 约为基准因子库的 **2 倍**
- 使用因子数量减少 **70%+**

### 2.2 自主模型优化（Model Optimization）

- **模型结构自动搜索**：LLM 提出模型架构创新
- **超参数自动调优**：迭代优化训练配置
- **因子-模型联合优化**：`fin_quant` 命令同时优化因子和模型

### 2.3 研报/论文自动提取与实现

- **金融研报因子提取**：自动阅读研报，提取因子公式并实现
- **论文模型复现**：自动阅读学术论文并实现模型结构
- 支持 PDF/URL 输入

### 2.4 数据科学 Agent / Kaggle Agent

- **自动特征工程**：在 Kaggle 竞赛中自动构建特征
- **自动模型调优**：迭代优化竞赛模型
- **MLE-Bench 第一名**：在 OpenAI 的 ML 工程能力评测中排名最高

### 2.5 RD-Agent(Q) — 多 Agent 量化框架

2025 年最新发布的 **RD-Agent-Quant** 是该项目的量化专精版本：

| 特性 | 说明 |
|------|------|
| **多 Agent 协作** | Research Agent（提出假设） + Development Agent（实现验证） |
| **因子-模型联合优化** | 交替优化因子挖掘和模型结构 |
| **数据中心设计** | 以数据质量驱动模型改进 |
| **深度集成 Qlib** | 使用 Qlib 作为底层回测和数据平台 |

### 3.1 核心模块说明

| 模块 | 路径 | 职责 |
|------|------|------|
| `rdagent/core/` | 核心框架 | 提案、实验、评估、进化 Agent、知识库 |

| `rdagent/scenarios/data_science/` | 数据科学 | Kaggle 竞赛、通用 ML 场景 |

| `rdagent/app/` | 应用入口 | CLI 命令入口、用户界面 |
| `rdagent/log/` | 日志 | 实验日志、Streamlit 可视化 |

### 3.2 进化循环（Evolving Loop）

RD-Agent 的核心机制是 **提出假设 → 实现 → 验证 → 学习** 的闭环：

```
┌────────────────────────────────────────────────┐
│              进化循环 (Evolving Loop)            │
│                                                │
│  1. Proposal Phase (提出假设)                    │
│     LLM 分析知识库 → 提出新因子/模型假设          │
│                   ↓                             │
│  2. Development Phase (开发实现)                 │
│     LLM 生成 Python 代码 → Docker 沙箱验证       │
│                   ↓                             │
│  3. Evaluation Phase (验证评估)                   │
│     Qlib 回测 → 计算 IC/ICIR/收益指标            │
│                   ↓                             │
│  4. Learning Phase (知识积累)                     │
│     更新知识库 → 指导下次迭代                      │
│                   ↓                             │
│  自动重复 N 轮直到收敛                            │
└────────────────────────────────────────────────┘
```

## 4. 使用方式

### 4.1 量化因子挖掘
```bash
# 因子 + 模型联合优化
rdagent fin_quant

# 仅因子挖掘
rdagent fin_factor

# 仅模型优化
rdagent fin_model
```

### 4.2 研报因子提取
```bash
# 从金融研报提取因子
rdagent fin_factor_report --report-folder=<研报目录>
```

### 4.3 论文模型复现
```bash
# 自动阅读论文并实现模型
rdagent general_model "https://arxiv.org/pdf/2210.09789"
```

### 4.4 Kaggle 竞赛
```bash
rdagent data_science --competition tabular-playground-series-dec-2021
```

## 5. 与 Qlib 的关系

RD-Agent 与 Qlib 是 **深度绑定** 的关系：

| 维度 | 说明 |
|------|------|
| **因子实现** | 生成的因子代码遵循 Qlib DataHandler 格式 |
| **模型实现** | 生成的模型代码遵循 Qlib Model 接口 |
| **回测验证** | 所有验证在 Qlib 回测框架中运行 |
| **数据依赖** | 使用 Qlib 的二进制数据存储 |
| **Docker 沙箱** | 代码在预装 Qlib 的 Docker 容器中执行 |

本质上：**RD-Agent 是 Qlib 的"AI 研究员"扩展**——Qlib 提供基础设施，RD-Agent 在其上自动化研究过程。

## 7. 局限性

| 局限 | 说明 |
|------|------|
| ⚠️ 仅支持 Linux | 依赖 Docker 执行沙箱，不支持 macOS/Windows |
| 💻 资源消耗大 | 需要 LLM API 调用（GPT-4/o1 级别效果最佳） |
| 📦 依赖 Qlib | 量化场景强绑定 Qlib 生态 |
| 🔧 复杂度高 | 配置和运行门槛较高 |
| 🇨🇳 A 股支持有限 | 默认场景偏向美股/国际市场数据 |
| ⏱️ 运行时间长 | 一轮完整进化可能需要数小时 |
| 🎯 因子粒度 | 产出因子仍偏学术化，实盘适用性需验证 |
